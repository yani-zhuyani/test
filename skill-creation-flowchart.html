Phase 0 · 需求发现

  • 发现约 70% 的 B 端规范是快手专有约定
  • 无 Skill 时设计稿屡被 UI 评审打回

Phase 5 · 差距分析
对比社区 Top 3 Skill，识别能力缺口
搜索社区中 B 端设计规范相关的 Top 3 Skill，与本地 date-design-guidelines 进行多维度对比，发现 5 类结构性缺口。
  • design-consistency-auditor（95 installs）— 审查机制
  • axiom-hig（138 installs）— HIG 原则 + 触发描述
  • design-system（15 installs）— 分阶段工作流
  • 缺口 1：无输出后自检 Checklist
  • 缺口 2：无具体 Design Token（颜色 / 间距 / 字体数值）
  • 缺口 3：页面级布局模式覆盖太窄
  • 缺口 4：无分阶段设计工作流
  • 缺口 5：触发描述精准性不足
