什么是 Takumi？
Takumi（/tɑːˈkuːmi/）是快手自研的 Code Agent， 类似社区的 Claude Code / Gemini-cli / Aider。你用自然语言给它派活， 它能理解上下文、调用工具、与大模型交互来完成任务。
默认支持 Kwaipilot 等快手内网模型，内置 Kmi / MUI 私域知识，目前处于内测阶段。

完整安装流程
Step 01 · 安装 Takumi
⚡ 快手内网
安装 Takumi CLI（Node.js 18+ 前提）
Takumi 发布在快手内网 npm 源，安装前必须先切换 registry。 安装完成后可以用 t 或 takumi 启动。 
# 第一步：切换到快手内网 npm 源（必须先做，否则找不到包）
npm set registry https://npm.corp.kuaishou.com/

# 第二步：全局安装
npm install -g @takumi/cli

# 验证安装：以下任一命令均可启动
t
takumi
📖 官方文档：Hello Takumi  ·  内测群可在文档底部扫码加入  ·  需要内网权限访问
Step 02 · 安装 figma-cli
✓ 成功
克隆 figma-cli 并写入快捷指令
figma-cli 是开源工具（MIT 协议），通过 WebSocket + Figma Plugin API 实现对画布的完整读写控制。 安装后创建 TAKUMI.md（从 CLAUDE.md 复制）， Takumi 会自动读取它，了解所有 figma-cli 命令规范。 
# 克隆到本地
cd ~ && git clone https://github.com/silships/figma-cli.git
Cloning into 'figma-cli'... 1048 objects received ✓

# 安装依赖
cd figma-cli && npm install
found 0 vulnerabilities ✓

# 配置快捷指令
npm run setup-alias && source ~/.zshrc
✓ Added fig-start alias to ~/.zshrc

# 关键：为 Takumi 创建规则文件（Takumi 自动读取 TAKUMI.md）
cp CLAUDE.md TAKUMI.md
💡 为什么要创建 TAKUMI.md？

Takumi 的 Rules 文件叫 TAKUMI.md（Claude Code 叫 CLAUDE.md，Cursor 叫 .cursorrules）。 figma-cli 项目自带了 CLAUDE.md，里面记录了所有命令语法和最佳实践。 复制一份改名后，Takumi 就能在每次启动时自动读取，无需手动解释如何控制 Figma。

Step 03 · macOS 权限授权
⚡ Takumi 解决
解决 macOS 完整磁盘访问权限问题
figma-cli 的 Yolo Mode 需要修改 Figma 的 app.asar 来开启 CDP 调试端口， 但 macOS 默认拒绝此操作。通过 Takumi（快手内部工具）完成授权， 是整个流程能跑通的关键节点，避免了手动进入「系统设置 → 隐私与安全性」的繁琐操作。 
# 未授权时直接运行会报：
✖ Setup failed
   EACCES: permission denied, open '.../Figma.app/Contents/Resources/app.asar'

# 通过 Takumi 完成授权后，该错误消除
（Takumi 辅助为 Terminal 授予「完整磁盘访问权限」）
Takumi 作为快手内部 Code Agent，具备工程化的权限管理能力。 如遇问题也可手动操作：系统设置 → 隐私与安全性 → 完整磁盘访问权限 → 添加 Terminal，然后重启终端。
Step 04 · 给 Figma 安装插件
✓ 一次性操作
在 Figma Desktop 中导入 FigCli 本地插件
Safe Mode 的通信机制：figma-cli 启动本地 Daemon → Figma 插件连接 Daemon 的 WebSocket :3456 → 插件代理执行所有画布 API。 需要在 Figma 中一次性导入本地 manifest，之后每次会话只需点击运行插件即可。 
# Figma Desktop 操作路径（确保已打开一个设计文件，能看到画布）
① 点击左上角 F 标志（Figma 主菜单）
② Plugins → Development → Import plugin from manifest...
③ 在文件选择弹窗按 Cmd + Shift + G，输入路径：
   /Users/zhuyani/figma-cli/plugin/manifest.json
④ 点击「打开」，导入完成

# 验证：Plugins → Development 下出现：
FigCli  [Development]  /Users/zhuyani/figma-cli/plugin ✓
💡 右键 FigCli → "Add to toolbar" 可固定到顶部工具栏，每次一键启动无需找菜单。
Step 05 · 连接 + 使用
✓ 全部打通
启动 Daemon → 激活 Figma 插件 → 用 Takumi 画设计稿
关键：必须先启动 Daemon，再在 Figma 里点击运行插件（顺序不能反）。 握手成功后，在 Takumi 里用中文描述需求，它会自动调用 figma-cli 命令在画布上生成设计。 
# 终端 窗口 1：启动 Daemon（此窗口需保持运行，不要关闭）
cd ~/figma-cli && node src/index.js connect --safe
✓ Daemon running in Safe Mode
  等待 Figma 插件连接...

# 在 Figma 中：Plugins → Development → FigCli（点击运行）

# 终端显示握手成功
✓ Plugin connected!
✓ Ready! Safe Mode active.

# 终端 窗口 2：启动 Takumi（在 figma-cli 目录下，会自动读取 TAKUMI.md）
cd ~/figma-cli && takumi

# 在 Takumi 里用自然语言下达指令：
"帮我在当前 Figma 画布上生成一个登录弹窗，包含邮箱、密码输入框和登录按钮"
"帮我加一套 shadcn 风格的设计令牌（Light/Dark 模式）"
"把当前选中的 Frame 导出为 CSS 变量"
🧠 Takumi 使用技巧

复杂设计任务可以先按 Shift + Tab 进入 Plan Mode，让 Takumi 先规划再执行，效果更好。
切换模型用 -m 参数，例如：takumi -m=gemini 使用 Gemini 模型进行 CR。
用 t commit 可以让 AI 自动生成规范的 Git 提交信息。

📋 每次开机后的使用流程（4 步）

1
打开 Figma Desktop，进入一个具体的设计文件（要能看到画布，不能停在首页）
2
终端运行：cd ~/figma-cli && node src/index.js connect --safe，等待提示
3
在 Figma 中点击：Plugins → Development → FigCli，等终端出现 Plugin connected!
4
新开终端，运行 cd ~/figma-cli && takumi，用中文告诉 Takumi 要做什么设计稿
