设计走查平台 — 系统架构

整体分 4 个模块，核心流程：输入 → 截图 → Diff → 报告

FRONTEND — 设计师侧
📋 走查任务创建
填 Figma URL + 页面 URL + 视口
📊 对比报告查看
左设计稿 / 右实现 / diff 高亮叠加
📝 问题标注 & 导出
标注问题，生成给前端的反馈清单
📈 走查历史 & 趋势
各页面还原分数走势，回归对比
BACKEND — 核心引擎
🎭 Playwright 截图服务
Headless 浏览器，指定 URL + 视口截全屏
🎨 Figma API 取图
解析 Figma URL → 拉取对应 Frame 导出 PNG
🔍 图像 Diff 引擎
pixelmatch 逐像素对比，生成差异热力图
🤖 AI 问题归因（可选）
Vision 模型分析差异区域，给出文字描述
OUTPUT — 报告推送
🌐 HTML 可视化报告
💬 PR Comment（GitHub/GitLab）
🔔 飞书 / 钉钉卡片推送
TECH STACK — 技术选型
Frontend: Next.js + React
Backend: Node.js + Express
截图: Playwright
Diff: pixelmatch / jimp
DB: SQLite（轻量，单机部署）
