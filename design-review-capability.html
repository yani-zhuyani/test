三层能力详解
LAYER 1
✅ 可实现
👁
视觉走查
对比设计稿截图与前端实现，量化还原度偏差
输入来源
规范 Skill 规则 Figma URL 页面 URL
点击展开步骤 ▾
1
获取规范基准
从规范 Skill 加载 Token 映射表（颜色/间距/字号期望值），作为走查标准
spec.getTokens() → rules.json
2
Figma 导出 + Playwright 截图
并行获取设计稿与前端截图，jimp 对齐尺寸
3
pixelmatch 生成热力图
逐像素对比，生成差异热力图 + 偏差坐标列表
4
AI Vision 归因（规范驱动）
Vision 模型分析差异区域，对照规范 Token 表判断：「实际值 vs 规范期望值」
5
输出问题清单（含规范引用）
每条问题标注对应规范条目，方便开发直接查阅
📐
规范 Skill 注入点：步骤 1 加载 Token 映射表；步骤 4 归因时对照规范值（非单纯 Figma 颜色），即使 Figma 设计稿本身有误，规范值优先。
Playwright pixelmatch jimp Claude Vision Figma REST API
LAYER 2
🚧 Beta
🖱
交互走查
按剧本执行操作，验证每个交互状态的视觉表现完整性
输入来源
规范状态定义 YAML 剧本 Figma Frames
点击展开步骤 ▾
1
规范 Skill 生成剧本框架
从规范 Skill 读取组件的状态枚举（hover/focus/error/loading…），自动生成 YAML 骨架，设计师只需填入 selector
spec.getStates('Button') → yaml 骨架
2
Playwright 执行操作序列
click / type / hover / scroll，每步截图存档
3
与 Figma Frame + 规范值对比
对比截图 + 检查规范（如 hover 颜色是否符合规范而非 Figma 里写错的值）
4
输出缺失状态清单 + GIF
哪些状态未实现，对应规范中的必须项 vs 可选项
📐
规范 Skill 注入点：步骤 1 自动生成 YAML 骨架（减少手写），步骤 3 以规范值为判断标准。规范里定义的「必须态」缺失 = P0，「建议态」缺失 = P1。
Playwright ffmpeg YAML DSL
LAYER 3
📋 规划中
🤖
AI 自动修复
定位源码 → 用规范正确值生成 patch → 截图验证 → 提 PR
输入来源
规范正确值 L1 问题清单 代码仓库
点击展开步骤 ▾
1
问题分级（规范驱动）
确定性问题 = 规范 Skill 中有明确正确值；主观问题 = 规范中无精确定义。前者自动修，后者人工处理
2
AST 定位源码
grep → tree-sitter → 定位目标属性的 CSS 声明行
支持：CSS / SCSS / Tailwind / CSS-in-JS
3
生成 patch（引用规范 Token）
直接使用规范 Token 变量名，不还原 Figma 魔法值。修复质量更高、更可维护
color: #6b6f80 → var(--color-text-secondary)
4
截图验证（最多3次）
重跑 L1 + 规范合规检查，还原分提升 ≥5 分且规范检查通过则 PR
5
提 PR + 飞书通知
PR 描述注明「依据规范条目 X.X.X 修改」，设计师一键确认
📐
规范 Skill 核心价值：步骤 3 的 patch 直接引用 Token（而非 Figma 颜色值），修复结果符合设计系统，避免魔法值扩散。规范即修复标准。

规范 Skill 贯穿全流程（步骤 0, 3, 4, 5）
步骤 3 分级：规范有明确定义 → 自动修；否则 → 人工问题单
步骤 5 双重验证：像素还原分 + 规范合规检查
问题处理决策规则（规范驱动版）
问题类型 来源层 规范 Skill 有定义 处理路径 规范注入方式
颜色值偏差L1✅ 有L3 自动修复从规范读 Token → var(--color-xx) 替换
间距偏差L1✅ 有L3 自动修复从规范读标准 px → patch spacing
字号/字重偏差L1✅ 有L3 自动修复从规范读字阶表 → patch font-size/weight
Hover/Focus 缺失L2✅ 有L3 半自动（需确认）规范定义 hover 颜色 → AI 生成 :hover 规则
Error/Empty 状态缺失L2⚠ 部分规范有 → L3；无 → 人工规范定义错误色/空态文案 → 提供参考
布局结构差异L1❌ 无人工处理—（主观判断，规范不覆盖）
组件使用错误L1✅ 有规范指出正确组件名规范的组件清单 → 标注应使用哪个组件
动效时长偏差L2⚠ 部分规范有定义 → L3规范的动效时长规范 → patch transition-duration
响应式断点异常L1❌ 无人工处理—（场景复杂，需人工分析）
内容截断/换行L1❌ 无人工处理—（内容相关，非样式规范）
💡
关键规则：「规范 Skill 有明确定义」= 可确定正确值 → 进 L3 自动修复。「规范未覆盖」= 主观判断 → 只输出问题单，由设计师和前端协商解决。规范 Skill 的覆盖度越高，自动修复率越高。
