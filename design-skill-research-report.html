shadcn/ui Skill
shadcn · 官方 Skill · 使用最广
最接近 B 端场景的设计系统 Skill。核心创新是动态注入项目实际状态，而非静态文档灌输。
动态注入 Pattern 强制 项目感知
★
每次对话运行 shadcn info --json，注入框架版本、已安装组件、alias 路径等实时配置
✓
强制 Pattern 组合：表单必须用 FieldGroup，选项必须用 ToggleGroup
✓
AI 先执行 shadcn docs 查找组件再生成代码，不靠记忆输出
🥈
brand-guidelines Skill
Anthropic · 官方 Skill · 品牌规范标杆
最标准的精确数值驱动规范 Skill。所有设计 Token 直接以 hex 值定义，无模糊描述。
精确数值 回退方案 技术绑定
★
不说"使用品牌色"，直接给 #d97757、#141413，可直接用于代码
✓
字体有明确 fallback：Poppins → Arial / Lora → Georgia
✓
规范直接对应 python-pptx 的 RGBColor 实现，降低技术应用门槛
🥉
frontend-design Skill
Anthropic · 官方 Skill · 反约束法
最强调负向约束的 Skill。明确告诉 AI 什么不能做，比"应该做什么"更有效。
禁止列表 决策前置 反 AI 味
★
禁止：Inter/Roboto 字体、紫色渐变、过度圆角 —— 明确写"不能用"比"应该用"落地更稳
✓
强制决策流程：先选 Tone → 先选 Color → 再写代码，防止随意生成
✓
Purpose / Tone / Constraints / Differentiation 四象限设计前置分析

核心方法论解析
三个 Skill 各自解决了什么根本性问题
🥇 shadcn/ui — 动态注入架构
Project Detection：检测到 components.json 时激活
Context Injection：运行 shadcn info --json 注入实时项目状态
Pattern Enforcement：规定必须用哪种组件组合
Component Discovery：生成前先查文档，不靠记忆
🥈 brand-guidelines — 精确数值架构
Token 集中定义：SKILL.md 顶部直接列出所有 hex 值
技术实现绑定：规范直接对应代码实现方式
Fallback 方案：环境不支持时的回退策略明确写出
无模糊描述：数值 > 语言描述，不存在解释空间
🥉 frontend-design — 负向约束架构
Forbidden List：明确禁止的字体、颜色、样式
Decision Flow：强制先做设计决策再写代码
Anti-pattern 警告：列出 AI 最容易犯的"AI 味"错误
Purpose 前置：写代码前强制分析用途、调性、约束

升级路线图
基于调研结论，按优先级排列的具体行动项
🔴
P0 — 在 SKILL.md 顶部加入 Core Design Tokens 章节
向 brand-guidelines 学习：将最常用的设计 Token 集中放在 SKILL.md 最顶部，精确到 hex 值和 px 数值，让 AI 无需翻 references 就能直接执行。例如：主色 #2563F4、弹框宽度 400px、按钮高度 32px、表单标签间距等。
学自 brand-guidelines
🔴
P0 — 增加 Forbidden Patterns（禁止模式）章节
向 frontend-design 学习：加一个专门的负向约束清单，明确写出 AI 最容易犯的错误。例如：「表格禁止使用 border-collapse，必须用规定 class」「胶囊标签页禁止用于上级导航」「弹框宽度禁止超过 400px」。负向约束比正向指引更难被 AI 忽略。
学自 frontend-design
🟠
P1 — 增加 Required Patterns（必须组合）清单
向 shadcn/ui 学习：为每种组件场景指定必须使用的组合方式。例如：「表单中多个输入框必须用 Form 组件包裹」「包含描述/备注的字段必须用 textarea 而非 input」「分页组件必须包含：总条数 + 页码 + 跳转 + 条数切换 四要素」。
学自 shadcn/ui
🟠
P1 — 在快速决策指南中增加弹框宽度速查项
当前 SKILL.md 快速决策指南覆盖了「选哪种组件」，但缺少「组件的关键数值」速查。建议为弹框（400px）、表格列宽、按钮高度等最常被忽视的数值规范，直接写入快速决策指南区域，不需要跳转 references 才能找到。
综合 Eval-1/3 高频失分项
🟡
P2 — 为高频场景添加完整代码示例
向 shadcn/ui 的「Ask AI to add a login form」模式学习：在 SKILL.md 或 references 中增加「新建弹框」「知识库列表+分页」「表格+筛选器」等完整代码示例，让 AI 在面对高频需求时可以直接参考正确的完整实现。
学自 shadcn/ui Pattern Discovery

核心结论
你的 Skill 优势

20+ 组件覆盖、完整决策树、内置禁止项、100% 实测合规率 —— 这些特性综合来看已经超过了三个标杆 Skill 中的任意一个。

一句话结论
你的 Skill 在广度（覆盖组件数、决策树完整性）上已处于行业领先，与 Top 3 的唯一差距在强度 —— 规范更多是「文档描述」形式，缺少 shadcn/ui 的「Pattern 强制执行」和 brand-guidelines 的「数值精确注入」。两个 P0 改动即可大幅提升。
