## 一、设计系统概述

**Material Design** 是 Google 开发的一套综合性设计系统，它通过统一的视觉语言和交互模式，为跨平台（Android、iOS、Web）的产品提供一致的用户体验框架。

### 设计系统的核心目标

- **统一性**：在不同平台和设备上提供一致的视觉语言
- **灵活性**：通过 Material Theming 支持品牌个性化定制
- **可访问性**：内置无障碍设计规范，面向所有用户
- **规模化**：模块化组件体系支持产品快速扩展

### 系统构成

```
Material Design 系统
├── 视觉基础（Foundations）
│   ├── 环境与表面（Environment/Surfaces）
│   ├── 布局（Layout）
│   ├── 颜色（Color）
│   ├── 字体排版（Typography）
│   ├── 图标（Icons）
│   └── 形状（Shape）
├── 动效（Motion）
│   ├── 理解动效（Understanding Motion）
│   └── 速度与缓动（Speed & Easing）
├── 交互模式（Interaction）
├── 组件（Components）
└── 无障碍与可用性（Usability）
    └── 无障碍（Accessibility）
```

## 二、核心设计原则

Material Design 的设计哲学基于**真实世界的物理规律**，将数字界面类比为现实中的材料和空间。

### 2.1 材料隐喻（Material Metaphor）

> "Material is the metaphor."

Material Design 的灵感来源于纸张和墨水。数字界面中的元素像现实世界中的物理材料一样运行：
- **表面具有物理属性**：有厚度（1dp）、有阴影、有层叠关系
- **遵循物理规律**：元素的运动、变化要符合自然世界的逻辑
- **光线创造深度**：通过光源和阴影表现层次关系

### 2.2 大胆、图形化、有意图（Bold, Graphic, Intentional）

- 版式、网格、空间、比例、颜色和图像不仅是视觉装饰，更是**传达层级、含义和焦点**的工具
- 有意图地强调关键行为，引导用户注意力
- 视觉设计应服务于功能，而非纯粹为了美观

### 2.3 有意义的动效（Motion Provides Meaning）

- 动效不只是视觉效果，它**解释了界面的变化和关系**
- 通过连续性、反馈和空间感，引导用户理解当前状态
- 动效应该是**有目的、合适且有意义**的，而非随意的装饰

## 三、视觉规范 — 颜色系统

### 3.1 颜色系统概述

Material 的颜色系统是将颜色系统地应用到 UI 的有组织方法。颜色角色有语义化名称，并在组件中有明确的使用定义。

### 3.2 颜色角色

| 颜色角色 | 用途说明 |
| -------- | -------- |
| **Primary**（主色）| 品牌核心颜色，用于关键 UI 组件如 FAB、按钮、激活状态 |
| **Secondary**（辅助色）| 用于强调、次要行为，提供品牌差异化 |
| **Surface**（表面色）| 用于卡片、列表等容器表面 |
| **Background**（背景色）| 用于整体页面背景 |
| **Error**（错误色）| 用于错误状态和错误提示组件 |
| **On [Color]**（前景色）| 叠加在对应颜色上的元素颜色（确保可读性） |

### 3.3 颜色配对规则

每种颜色都有对应的"On 颜色"用于叠加其上的元素（文字、图标等）：

```
Primary   → On Primary
Secondary → On Secondary
Surface   → On Surface
Background → On Background
Error     → On Error
```

> **原则**：当 "Primary" 颜色作为容器背景时，"On Primary" 颜色用于其内部内容，以保证一致性和可访问性对比度。

### 3.4 颜色调色板结构

每种颜色从 50（最浅）到 900（最深）有完整的色阶：

```
颜色色阶：50 / 100 / 200 / 300 / 400 / 500 / 600 / 700 / 800 / 900
```

**颜色调和类型：**
- **主色 + 互补色**（Complementary）
- **主色 + 类似色**（Analogous）
- **主色 + 三角色**（Triadic）

### 3.5 颜色可访问性

- 确保文本与背景之间的**对比度符合 WCAG 标准**
- 正常文本最低对比度：**4.5:1**
- 大号文本最低对比度：**3:1**
- 切勿仅通过颜色传递信息，还需配合图标、文本等辅助标识

## 四、视觉规范 — 字体排版

### 4.1 字体比例系统

Material Design 提供 **13 种字体样式**，覆盖从标题到正文的所有场景：

| 样式名称 | 典型用途 |
| -------- | -------- |
| **H1** | 最大标题，用于短文本或数字，可用表现力字体 |
| **H2** | 次大标题 |
| **H3** | 三级标题 |
| **H4** | 四级标题 |
| **H5** | 五级标题 |
| **H6** | 六级标题 |
| **Subtitle 1** | 中等强调的短文本，支持衬线/非衬线字体 |
| **Subtitle 2** | 较小的副标题 |
| **Body 1** | 主要正文内容 |
| **Body 2** | 次要正文内容 |
| **Button** | 按钮文本，通常大写 |
| **Caption** | 图片说明、次要信息 |
| **Overline** | 用于分类标签，小号全大写 |

### 4.2 字体属性规范

**每种字体样式的关键属性可定制：**
- **字体族**（Typeface）：如 Roboto、品牌字体等
- **字重**（Font Weight）：Regular / Medium / Bold 等
- **字母大小写**（Letter Case）：正常 / 全大写
- **字间距**（Letter Spacing）

### 4.3 字体选择原则

- **标题（Headlines）**：可选择表现力强的字体，如展示体、手写体或脚本体
- **副标题（Subtitles）**：衬线或非衬线字体均可
- **正文（Body）**：应选择可读性强的字体，避免过于装饰性的字体
- **按钮/标签**：清晰、易识别的字体

## 五、视觉规范 — 形状

### 5.1 形状的作用

形状样式可以帮助：
- **引导注意力**：指向组件或识别组件
- **传达状态**：通过形状变化表现交互状态
- **表达品牌个性**：圆角、切角等形状风格传递品牌特征

### 5.2 形状分类

所有 Material 组件根据尺寸分为三类：

| 形状类别 | 适用组件示例 |
| -------- | ------------ |
| **Small**（小型）| Chip、FAB（小）、Text Field 等 |
| **Medium**（中型）| Card、Dialog、FAB（中）等 |
| **Large**（大型）| Bottom Sheet、Navigation Drawer 等 |

### 5.3 形状定制规则

- 同一形状类别的所有组件可共享一个形状样式
- 形状样式包含：角的样式（圆角/切角）+ 角的尺寸
- 通过形状定制工具可以快速为整个产品生成统一形状方案

## 六、环境与表面规范

### 6.1 Material 环境

Material Design 的 UI 在一个**三维空间**中呈现，通过光线、表面和阴影表达深度。

**三维坐标系：**
- x 轴（水平方向）
- y 轴（垂直方向）
- **z 轴**（深度/海拔高度）

### 6.2 表面属性

Material 表面具有以下**固定且不可改变**的特征：

| 属性 | 规格 |
| ---- | ---- |
| 厚度 | 始终为 **1dp**（不可更改） |
| 尺寸 | x/y 维度可自由变化（单位：dp） |
| 分辨率 | **无限分辨率** |
| 内容显示 | 可展示任意形状和颜色，内容不增加厚度 |

### 6.3 物理属性（重要规则）

**表面是固体，遵循以下规则：**

| 规则 | 描述 |
| ---- | ---- |
| ✅ 用户输入只影响表面顶部 | 输入事件不能穿透材料 |
| ✅ 同一位置只能有一个表面 | 多个 Material 元素不能同时占据同一空间点 |
| ✅ Material 不能穿过其他 Material | 改变海拔时不能穿透其他表面 |
| ❌ Material 不像气体 | 进入/退出通过透明度/尺寸/位置变化实现 |
| ❌ Material 不像液体 | 虽可展示流动内容，但本身非流动物质 |

### 6.4 表面变换规则

- **可以**：改变形状
- **可以**：改变透明度（整体或局部）
- **可以**：沿平面方向增大或缩小
- **禁止**：弯曲或折叠超过 UI 深度范围
- **可以**：多个表面合并为一个
- **可以**：分裂后再合并

### 6.5 表面行为类型

| 类型 | 描述 |
| ---- | ---- |
| **Rigid（刚性）** | 在所有交互中保持尺寸不变 |
| **Stretchable（可拉伸）** | 可以沿一个或多个边缘增长或收缩，达到限制后变为刚性 |
| **Pannable（可平移）** | 保持尺寸不变，内容可在其上滚动 |

### 6.6 Scrim（遮罩）

Scrim 是一种临时处理，可以使表面上的内容变得不那么突出：
- 暗化或亮化表面及内容
- 降低表面和内容的透明度
- **不要**使用完全透明的表面，因为边界不清晰

### 6.7 阴影与层级

- 不同海拔高度的表面投射不同大小的阴影
- **阴影必须对应实际海拔**，不能仅用颜色模拟阴影
- 基础 Material 表面：不透明白色，1dp 厚度，投射阴影

## 七、布局规范

### 7.1 布局三原则

| 原则 | 说明 |
| ---- | ---- |
| **可预测（Predictable）** | 使用直观可预测的布局，具有一致的 UI 区域和空间组织 |
| **一致（Consistent）** | 布局应一致地使用网格、基准线和内边距 |
| **响应式（Responsive）** | 布局是自适应的，响应用户、设备和屏幕元素的输入 |

### 7.2 布局结构解剖

大屏幕布局包含三个主要区域：

```
┌──────────────────────────────────────┐
│              App Bar (1)             │
├─────────────┬────────────────────────┤
│             │                        │
│ Navigation  │       Body             │
│    (2)      │        (3)             │
│             │                        │
└─────────────┴────────────────────────┘
```

**各区域说明：**
- **App Bar**：展示主要操作和元素，帮助执行主要任务
- **Navigation**：展开时固定宽度 256dp，折叠后为 72dp
- **Body**：展示应用大部分内容（列表、卡片、按钮、图像等）

### 7.3 响应式列网格

列网格由**列（Columns）+ 沟槽（Gutters）+ 外边距（Margins）**组成。

**各断点网格规范：**

| 设备类型 | 屏幕宽度 | 外边距 | Body | 列数 |
| -------- | -------- | ------ | ---- | ---- |
| 极小（手机） | 0-599dp | 16dp | 流式 | **4列** |
| 小（平板） | 600-904dp | 32dp | 流式 | **8列** |
| 小（平板） | 905-1239dp | 流式 | 840dp | **12列** |
| 中（笔记本） | 1240-1439dp | 200dp | 流式 | **12列** |
| 大（桌面） | 1440dp+ | 流式 | 1040dp | **12列** |

### 7.4 布局组合规则

#### 视觉分组（Visual Grouping）
相似内容或功能的元素可以通过**留白、排版和分割线**分组，并与其他元素隔离。

#### 包含关系（Containment）
上下文相关的元素可通过"包含"概念更紧密地组合：
- **隐式包含**：减少相关元素周围的空间，增大组外空间创建概念边界
- **显式包含**：添加轮廓或高程层级（如卡片）

#### 文字行长（Scaling with Text）
- **推荐每行文字 40-60 个字符**，保证可读性
- 超过最大行长时，考虑调整行高提升可读性

#### 锚点与约束（Anchors and Constraints）
- 内部元素可锚定到左、右或居中
- 部分元素（如 FAB、导航抽屉）保持固定位置
- 组件的内部组合应适应其所在设备的人体工程学需求

### 7.5 Material 度量单位

| 用途 | 对齐基准 |
| ---- | -------- |
| 主要布局单位 | **8dp 网格** |
| 组件尺寸递增 | 8dp 的整数倍 |
| 小型元素（图标等） | **4dp 网格** |
| 字体基线间距 | **4dp 基线网格** |

## 八、动效与交互规范

### 8.1 动效的作用

动效用于**表达品牌的个性和风格**，同时实现以下功能：

| 功能 | 说明 |
| ---- | ---- |
| **引导注意力** | 过渡动效引导用户关注关键操作 |
| **传达变化** | 解释 UI 状态的变化和元素间的关系 |
| **提供反馈** | 对用户输入提供即时视觉响应 |
| **增强品牌感** | 定制化的过渡风格传递品牌特征 |
| **增添愉悦感** | 图标、插画的微动效增加趣味性 |

### 8.2 动效应用场景

- **过渡效果（Transitions）**：可定制以匹配品牌风格（如视差效果、强调缓动）
- **图标与插画动效**：为图标含义添加活力，增强品牌表达
- **奖励动效**：在用户完成任务的关键节点使用，增加记忆点

### 8.3 动效速度规范

#### 持续时间（Duration）规范

| 过渡类型 | 持续时间范围 | 示例 |
| -------- | ------------ | ---- |
| **关闭/退出（Exit）** | 较短 | 导航抽屉关闭：200ms |
| **打开/进入（Enter）** | 略长 | 导航抽屉打开：250ms |

#### 按过渡区域大小分类：

| 区域大小 | 持续时间 | 示例 |
| -------- | -------- | ---- |
| **小型**（图标、选择控件） | ~100ms | Switch 动画：100ms |
| **中型**（底部表、展开的 Chip） | 200-250ms | Bottom Sheet 展开：250ms，收起：200ms |
| **大型**（FAB → 全屏对话框） | 250-300ms | FAB 展开：300ms，收起：250ms |

**原则：** 过渡不能太快（用户无法跟随）也不能太慢（妨碍用户前进）。

### 8.4 缓动类型（Easing）

缓动调整动画的变化速率，让元素加速和减速，模拟自然世界中的运动感。

| 缓动类型 | 适用场景 | CSS cubic-bezier |
| -------- | -------- | ---------------- |
| **Standard（标准缓动）** | 最常见，元素起止均静止 | `cubic-bezier(0.4, 0.0, 0.2, 1)` |
| **Emphasized（强调缓动）** | 需特别强调动画末尾时，配合较长持续时间 | 仅 Android 支持 |
| **Decelerated（减速缓动）** | 元素**进入**屏幕时，从最大速度到静止 | `cubic-bezier(0.0, 0.0, 0.2, 1)` |
| **Accelerated（加速缓动）** | 元素**退出**屏幕时，从静止到最大速度 | `cubic-bezier(0.4, 0.0, 1, 1)` |

**各平台实现对应：**

| 缓动类型 | Android | Flutter | iOS |
| -------- | ------- | ------- | --- |
| Standard | `FastOutSlowInInterpolator` | `standardEasing` | `initWithControlPoints:0.4f:0.0f:0.2f:1.0f` |
| Decelerated | `LinearOutSlowInInterpolator` | `decelerateEasing` | `initWithControlPoints:0.0f:0.0f:0.2f:1.0f` |
| Accelerated | `FastOutLinearInInterpolator` | `accelerateEasing` | `initWithControlPoints:0.4f:0.0f:1.0f:1.0f` |

## 九、无障碍设计规范

### 9.1 无障碍文本

无障碍文本供屏幕阅读器（如 Android TalkBack、iOS VoiceOver、桌面 JAWS）使用，包含：

- **可见文本**：UI 标签、按钮文字、链接、表单文本
- **不可见替代文本**：图片的 alt 文本

### 9.2 Alt 文本（替代文本）规范

Alt 文本是将视觉 UI 转化为文字描述的关键，限制最大 **125 个字符**。

**规范要求：**

| 原则 | 说明 |
| ---- | ---- |
| ✅ **信息丰富** | 用简短短语表达图片内容，而非仅一两个词 |
| ✅ **简洁可读** | 屏幕阅读器可快速读完，给用户提供上下文 |
| ✅ **包含关键词** | 帮助用户理解图片所在位置背景 |
| ✅ **不加前缀** | 无需写"图片是..."或"图像是..."，屏幕阅读器自动说明 |
| ❌ **不过长** | 超过 125 字符会被截断 |
| ❌ **不堆砌关键词** | 仅为 SEO 堆砌词汇会混淆用户 |

### 9.3 图片文字说明层级

图片周围的三类文本：

```
┌─────────────────────────────────────┐
│         图片（Image）（1）           │
├─────────────────────────────────────┤
│      标注文字（Caption）（2）        │
│  解释图片上下文（谁/什么/何时/何地） │
├─────────────────────────────────────┤
│    相邻文本（Adjacent Text）（3）    │
│ 叙述性内容，可替代 alt 文本的作用    │
└─────────────────────────────────────┘
```

**各类文本使用规则：**
- **Caption（标注）**：解释图片上下文信息，面向所有用户；长描述应用 Caption 而非 alt text
- **Adjacent text（相邻文本）**：叙述性正文；如果已充分解释图片，alt 可留空（`alt=""`）
- **嵌入图片中的文字**：屏幕阅读器无法读取，必须在 alt text 中重复关键信息

### 9.4 Alt 文本与 Caption 的区别

| 类型 | 用途 | 长度限制 | 面向用户 |
| ---- | ---- | -------- | -------- |
| Alt text | 图片视觉描述（颜色、尺寸、位置） | 最多 125 字符 | 无法看到图片的用户 |
| Caption | 图片上下文（who/what/when/where） | 无固定限制 | 所有用户 |

> ⚠️ **注意**：Alt text 与 Caption 不要重复相同内容，否则会拖慢屏幕阅读器用户的阅读速度。

## 十、Material Theming 定制化

### 10.1 什么是 Material Theming

Material Theming 使品牌能轻松定制 Material Design，通过内置支持对颜色、字体样式和角形状进行自定义，使产品与品牌外观和感觉保持一致。

### 10.2 三大定制维度

#### 颜色定制（Color）

```
品牌调色板
  ├── 主色（Primary）——→ 按钮、FAB、选中状态
  ├── 辅助色（Secondary）——→ 强调元素、次要行为
  ├── 表面色（Surface）——→ 卡片、底部表
  ├── 背景色（Background）——→ 页面背景
  └── 错误色（Error）——→ 错误状态
```

#### 字体排版定制（Typography）

- 13 种全局字体样式均可自定义字体族、字重、字母大小写
- 可替换为品牌字体，同时保持系统可读性

#### 形状定制（Shape）

- 小型（Small）、中型（Medium）、大型（Large）三类形状分类
- 可批量修改某类别下所有组件的形状风格
- 使用**形状定制工具**生成自定义形状样式

### 10.3 定制化示例

Material 按钮（Button）可以通过以下方式匹配品牌：

| 属性 | 默认值 | 可定制为 |
| ---- | ------ | -------- |
| 颜色 | #6200EE | 品牌主色 |
| 字体 | Roboto Medium | 品牌字体 |
| 圆角形状 | 4dp 圆角 | 全圆角 / 直角 / 切角 |
